总结在使用Unity Addressable System过程中遇到的一部分问题，该篇主要记录Addressable在Unity编辑器的各种组织结构。本文Addressable后续简称AA。

Addressable安装

最简单的方式

打开Window - Package Manger，找到Addressable安装找一个合适的版本:

Addressable Version Seleclt.

找一个合适的版本

在Package Manager选择一个合适的Addressable版本还要结合Addressable更新log，打开Addressable更新日志，版本列表展示了Address able对应的Unity版本：

Addressable Version List.

Unity这里分了两个分支维护Addressable，其中一个分支是1.21版本对应的Unity版本是2019+，另一个分支是2.0版本对应的Unity版本是2023+:

Addressable Branch.

稍复杂的方式

打开Addressable Package Json，这里列明了每一个Addressable版本对应的Unity版本，结合Addressable更新日志找到一个Bug最少又符合对Unity版本要求的Addressable版本。

找到合适的Addressable版本后，打开Packages/manifest.json，新增依赖包，比如：”com.unity.addressables”: “1.21.19”。

安装Addressable依赖

Addressable Dependencies.

Addressable最重要的依赖是ScriptableBuildPipeline，SBP这个包要更新到1.21.23或2.1.x以上，旧版SBP对SpriteAtlas图集打包会有冗余。 打开Packages/manifest.json，强制指定SBP版本”com.unity.scriptablebuildpipeline”: “1.21.23”,

SBP指定版本.

Addressable结构预览

安装好了AA之后，会在Assets目录生成以下目录：

目录说明
AddressableAssetsData根目录也包含几个重要文件
Android打包后生成的平台目录
AssetGroups给某资源设置寻址地址并指定Group后的Group描述文件
SchemasGroup组成的逻辑结构和特性描述文件
AssetGroupTemplates生成Group的模版描述文件
DataBuilders自定义AssetBundle打包脚本目录

AddressableAssetsData根目录包含的重要文件：

  • AddressableAssetSettings：AA的构建AssetBundle设置。
  • DefaultObject：保存当前系统AA生成的GUID，该GUID绑定AddressableAssetSettings对象。
  • CacheInitializationSettings：AB缓存设置。
  • ProfileDataSourceSettings：AB保存和加载地址设置。

AddressableAssetSettings说明

AddressableAssetSettings预览.

Profile

Profile预览.

Profile In Use可以选择系统自带或自定义，其中自定义使用Manager按钮打开Profile Window:

Profile Window.

Profile主要作用是指定打包地址和加载地址。见Catalog

Catalog

Catalog设置.
  • Player Version Override: 这个参数会对构建好的catalog进行重命名，unity使用时间戳，我们自己可以用自己的构建版本号，格式可以是catalog_1.2.1720684396.json。这篇对catalog详解这里先按下不表。
  • Compress Local Catalog: 启用会将catalog.json压缩成catalog.bundle，显著减少文件size。
  • Build Remote Catalog: 要热更catalog就要启用。
  • Build Path: 构建Assetbundle的保存地址。
  • Load Path: 加载AssetBundle的地址，这里我是设置成相对地址，在InternalIdTransformFunc做完成加载地址组装。
  • Only Update catalogs manually: 建议启用，可以定制更新catalog方式。

Update a Previous Build

Update Downloads设置.

指定addressables_content_state.bin的保存地址，该文件只在本地构建时有用，有兴趣看[官方workflow](https://docs.unity3d.com/Packages/com.unity.addressables@2.2/manual/content-update-builds-overview.html。简单来说就是打开Groups-Tools-Check Content Update Restrictions窗口，选择上一次保存的addressables_content_state去比对catalog，生成一个新的热更bundle。

由于我开了手动更新catalog，同时自己维护了一套更新列表，就禁用了这条属性。

Downloads

略

Builds

Build设置.
  • Build Addressables on Player Build: 出包的时候一起出Assetbundle，我关了不需要。
  • Ignore Invalid/Unsupported Files in Build: 排除无效文件。
  • Unique Bundle IDs: Assetbundle唯一标识符。
  • Contiguous Bundles: 定义的Assetbundle内容布局，启用。
  • Non-Recursive Dependency Calculation: 循环依赖计算耗费时间，开启只计算直接依赖，启用。
  • Strip Unity Version From AssetBundles: 裁剪Assetbundle内的unity版本号，启用。
  • Disable Visible Sub Asset Representations: 不直接拖拽引用某个sprite，开启该项可以加快构建时间，开不开无所谓。

Build And Play Mode Scripts

Build Bundle设置.

play mode指定在编辑器是怎么加载资源，Use Existing Building可以强制跑bundle模式，Use AssetDatabase无视依赖直接加载资源。

Addressable Groups

Group是Addressable组织资源的一种容器，必须要把资源加入Group才能参与打包。

直接说使用结论

  • Group越少越好，在创建Group时非常耗时，成百上千个Group可能卡10分钟甚至更久。
  • Group越少意味着Asset Entry越多，但是没关系，还有label标签可以用。
  • label标签可以做分类划分，具有相同label标签的资源打成一个bundle包。
Group设置.

Content Packing & Loading是BundleAssetGroupSchema，Content Update Restriction是ContentUpdateGroupSchema，均继承自AddressableAssetGroupSchema。应该是都要添加到Group内的。这两个文件描述了一个Group构建时所需要的属性。

  • Include In Build: 启用才能构建该组内所包含的Asset Entry。
  • Include Addresses in Catalog: 禁用后构建bundle时不会包含在catalog.json。
  • Include GUIDs in Catalog: 使用AssetReference拖拽引用就得启用。
  • Include Labels in Catalog: 不通过label标签加载资源，我这里不使用label去加载资源。
